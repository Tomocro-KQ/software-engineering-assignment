## 2. Purpose of the Software
* **Software Development Process:** This project follows the **Waterfall Model**. [cite: 46]
* **Justification:** Since the requirements for a To-Do List (Create, Read, Update, Delete) are well-defined and unlikely to change, the Waterfall model ensures a structured and disciplined approach. [cite: 47]
* **Target Market:** Students and professionals who need a lightweight, offline desktop application to manage daily tasks. [cite: 48]

## 3. Software Development Plan
* **Development Process:** Requirements -> Design -> Implementation -> Testing -> Deployment. [cite: 50]
* **Members (Roles & Responsibilities):** * Lead Developer: Software Logic & Data Persistence. [cite: 51]
    * UI Designer: Tkinter Interface & Layout (Padding/Margin). [cite: 51]
* **Algorithm:** The application uses a JSON-based sorting algorithm to display tasks by priority level (High > Medium > Low). [cite: 53]
* **Current Status:** Pilot Level (Stable). [cite: 54]
* **Future Plan:** Integration with a cloud database for cross-device syncing. [cite: 55]
