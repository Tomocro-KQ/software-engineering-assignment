## 2. Purpose of the Software
* **Software Development Process:** This project follows the **Waterfall Model**.
* **Justification:** Since the requirements for a To-Do List (Create, Read, Update, Delete) are well-defined and unlikely to change, the Waterfall model ensures a structured and disciplined approach.
* **Target Market:** Students and professionals who need a lightweight, offline desktop application to manage daily tasks.

## 3. Software Development Plan
* **Development Process:** Requirements -> Design -> Implementation -> Testing -> Deployment.
* **Members (Roles & Responsibilities):** * Lead Developer: Software Logic & Data Persistence.
    * UI Designer: Tkinter Interface & Layout (Padding/Margin).
* **Algorithm:** The application uses a JSON-based sorting algorithm to display tasks by priority level (High > Medium > Low).
* **Current Status:** Pilot Level (Stable).
* **Future Plan:** Integration with a cloud database for cross-device syncing.

## 4. Environment & Declaration
* **Language:** Python 3.10
* **Libraries:** Tkinter (GUI), JSON (Storage).
* **Declaration:** This software was developed independently. No external third-party packages were used.
